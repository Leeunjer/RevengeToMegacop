# /document — 코드 설명 문서 생성

지정된 코드 파일을 읽고, 해당 파일이 위치한 디렉토리의 `Documents/` 하위 폴더에 마크다운 설명 문서를 생성한다.

## 대상 결정

1. `$ARGUMENTS`가 지정된 경우 → 해당 파일을 대상으로 한다
2. `$ARGUMENTS`가 비어 있는 경우 → 사용자에게 파일 경로를 물어본다

## 절차

1. 대상 파일을 Read 도구로 읽는다
2. 파일이 의존하는 다른 파일이 있으면 함께 읽어 맥락을 파악한다
3. 아래 작성 규칙에 따라 설명 문서를 작성한다
4. 대상 파일이 위치한 디렉토리에 `Documents/` 폴더를 생성한다 (이미 있으면 건너뜀)
5. `Documents/{클래스명}.md` 파일로 저장한다
6. 이미 같은 이름의 문서가 있으면 덮어쓴다

## 작성 규칙

### 문서 구조

```markdown
# {클래스명} 설명

이 클래스는 **"{한 줄 요약}"**을 구현한다.

---

## 전체 흐름
(단계별 흐름을 간단한 다이어그램 또는 화살표로 표현)

---

## {단계/섹션별 설명}
(각 주요 메서드나 로직 블록을 섹션으로 나누어 설명)

---

## Inspector 파라미터 요약 (해당되는 경우)
(SerializeField 파라미터를 표로 정리)
```

### 문체

- **한국어**로 작성한다
- 코드를 모르는 사람도 이해할 수 있는 **쉬운 말**로 쓴다
- 전문 용어를 쓸 때는 괄호 안에 간단한 설명을 덧붙인다
- 비유나 ASCII 다이어그램을 적극 활용한다
- 코드를 그대로 옮기지 않고 **동작 원리**를 설명한다
- 불필요하게 길어지지 않도록 핵심만 담는다

### 금지 사항

- 코드 전체를 문서에 복사하지 않는다
- 영어로 작성하지 않는다
- 줄 번호를 하드코딩하지 않는다 (코드 변경 시 틀어지므로)
